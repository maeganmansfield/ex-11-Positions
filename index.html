The parent list needs postion: relative so that the child submenu can be positioned with position: absolute absolute relative to it - not relative to the entire page.

The submenu (a nested ul inside one of the li elements) is hiddenby defult with display: none. When the user hovers over the parent list item, the submenu is shown with display: block.

The key concept: posittion: absolute places an element relative to initial-scale nearest ancestor that has position: relative. Without that ancestor, it positions relative to the page - which is almost never what you want.
